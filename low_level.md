# strict aliasing

Компилятор предполагает, что указатели разных типов (например `int*` и `float*`) не ссылаются на одну и ту же память.
Если это нарушается, поведение программы становится неопределённым.

```
int x = 42;
float* f = reinterpret_cast<float*>(&x);
*f = 3.14f;  // UB: обращаемся к int как к float
```
Оптимизатор может кэшировать `x` в регистре, решив, что `f` не влияет на него, и тогда `x` и `*f` рассинхронизируются.

**Разрешённые исключения:**
- Доступ через `char*` или `std::byte*` разрешён (для побайтового чтения).
- `std::bit_cast` безопасен, так как копирует, а не переинтерпретирует.

# alignment

Типы данных должны быть размещены в памяти по адресам, кратным их выравниванию. Например, `int` часто требует адрес, кратный 4.

```
char buf[4];
int* p = reinterpret_cast<int*>(buf + 1);  // адрес не кратен 4
*p = 10;  // может вызвать аппаратное исключение
```

Некоторые процессоры (например ARM) не могут читать/писать невыравненные адреса.

Решение:
- Использовать `std::bit_cast` или `memcpy`.
- Для структур — `alignas(T)` или `std::aligned_storage`.

# volatile

Квалификатор, указывающий компилятору, что значение переменной может изменяться извне, и оптимизации для неё нужно ограничить. То есть `volatile` говорит компилятору "эта переменная может меняться сама по себе, не через мой код". Поэтому компилятор не имеет права её кэшировать или оптимизировать.

Используется для:
1. Регистров устройств в драйверах. Например, программа читает регистр устройства, который обновляется процессором, таймером или внешним сигналом.
   ```
   volatile int* status = (int*)0x4000;  // адрес регистра устройства
   while (!(*status & 1)) { }            // ждём, пока устройство готово
   ```
2. Переменных, изменяемых аппаратно или другим потоком без синхронизации (в старом коде). Раньше `volatile` использовали, чтобы один поток мог "увидеть" изменения от другого. Сейчас для этого применяют `std::atomic`.
   ```
   volatile bool ready = false;
   // поток 1: ready = true;
   // поток 2: while (!ready) {}
   ```
   Это работает ненадёжно (нет синхронизации), но объясняет идею.
