# `std::map`, `std::set`
Основаны на сбалансированных двоичных деревьях поиска. Конкретнее — почти всегда это красно-чёрные деревья (red-black tree).

Благодаря этому:
- элементы хранятся в отсортированном порядке (по ключу или по `operator<`);
- все основные операции (`insert`, `erase`, `find`, `lower_bound`, `upper_bound`) работают за O(log n);
- итерация идёт в отсортированном порядке.

`map` хранит пары (ключ, значение). `set` хранит только ключи.

## ⚫🔴 Красно-чёрное дерево

Это BST с дополнительными правилами, где каждая вершина помечена либо красным, либо чёрным.

Правила красно-чёрного дерева:
- Каждый узел — красный или чёрный.
- Корень всегда чёрный.
- Листья-"нуль-ссылки" считаются чёрными.
- Красный узел не может иметь красного родителя ("красный не рядом с красным").
- Любой путь от корня до листа проходит через одинаковое число чёрных узлов.

Благодаря этим правилам дерево остаётся почти идеально сбалансированным: его высота не превышает $2\log_{2}(n+1)$.

# `std::unordered_map`, `std::unordered_set`
Основаны на хеш-таблицах (обычно — хеш + открытая адресация или хеш + цепочки списков). Элементы не имеют определённого порядка.

Основные операции (`insert`, `erase`, `find`) работают в среднем за O(1), но в худшем случае (при коллизиях) могут вырости до O(n).

Используют `std::hash<key>` для вычисления хеша ключа (по умолчанию).

## Хэширование
### Классы хэш-функций
1. Простые:
   - Остаток от деления: `h(k) = k % M`. Быстрый, но легко даёт много коллизий (например, все чётные числа в одну половину таблицы).
   - Умножение и модуль: `h(k) = (a * k + b) % p % M`, где `p` — большое простое. Лучше распределяет, часто используется в конкурсном программировании.
2. Для строк:
   - Полиномиальный rolling hash: `h(s) = (s[0]*p^0 + s[1]*p^1 + ... + s[n-1]*p^(n-1)) mod M`
   - Диспергирующие функции: djb2 (Дэн Бернстайн, популярна в C и Python до перехода на SipHash). sdbm (использовалась в старых UNIX-базах данных).
3. Криптографические (сильные, но медленные). MD5, SHA-1, SHA-256. Дают «равномерные» и устойчивые хэши. Но они слишком медленные. Их применяют, если нужно защита от атак по коллизиям (например, чтобы злоумышленник не заставил `unordered_map` деградировать в O(n)).
4. Современные быстрые небезопасные: MurmurHash, CityHash (Google), xxHash, FNV-1a (очень простая и быстрая, часто в системном коде). Эти функции дают очень хорошее распределение, быстро считаются и поэтому часто применяются в базах данных, компиляторах, системных библиотеках.

Для обычных контейнеров в C++ обычно хватает встроенного `std::hash`. Для больших строковых ключей (например, в базе данных или поисковом индексе) часто применяют MurmurHash или xxHash — они быстрые и качественные. Для безопасности (в веб-сервисах) используют SipHash (по умолчанию в Python dict и Ruby hash) → защищает от атак по коллизиям.

### Коллизии и их разрешение
Коллизия = два разных ключа попали в один индекс массива.
Основные методы разрешения:
- Цепочки (chaining) — в каждой ячейке таблицы хранится список/вектор элементов.
    ```
    [0] 
    [1] 
    [2] → 12 → 7
    [3] → 3
    [4]
    ```
- Открытая адресация — если ячейка занята, ищем следующую свободную (линейно, квадратично, двойное хеширование).
    ```
    [0] 
    [1] 
    [2] → 12
    [3] → 7 (поставили сюда, т.к. 2 занята)
    [4] → 3
    ```

### Перестроение хэш таблицы
Load factor (LF) показывает, насколько заполнена хэш-таблица:
$$LF = \frac{elements}{buckets}.$$
Каждый контейнер имеет максимальный коэффициент загрузки, который можно получить через `max_load_factor()`. По умолчанию в STL он обычно 1.0. Если LF > max_load_factor, то таблица перестраивается:
- количество корзин увеличивается (обычно примерно в 2 раза, иногда больше),
- все элементы ре-хэшируются в новые корзины.


# Объект как ключ
Для `std::map` и `std::set` ключи должны быть сравнимы для упорядочивания, т.е.:
- Оператор < (operator<) должен быть определён.
- Он используется для сортировки элементов.
- Можно определить свой компаратор вместо `<`, но по умолчанию — `<`.

Для `std::unordered_map` и `std::unordered_set` ключи должны быть хэшируемыми и сравнимыми на равенство, т.е.:
- Функция хеширования (std::hash<Key> или своя).
- Оператор равенства (`operator==`).
```
struct Point {
    int x, y;

    bool operator==(const Point& other) const {
        return x == other.x && y == other.y;
    }
};

// Пользовательская хеш-функция
struct PointHash {
    std::size_t operator()(const Point& p) const {
        return std::hash<int>()(p.x) ^ (std::hash<int>()(p.y) << 1);
    }
};

std::unordered_map<Point, std::string, PointHash> pointNames;
pointNames[{1,2}] = "A";
pointNames[{2,3}] = "B";
```
