# move семантика
move-семантика позволяет передавать ресурсы объекта вместо их копирования. Это особенно важно для объектов, которые управляют динамической памятью, файлами, сокетами и другими ресурсами.
```
std::string s1 = "Hello, world!";
std::string s2 = std::move(s1);  // s2 "забирает" данные у s1
```

# lvalue, rvalue
## Lvalue
lvalue ("locator value") — объект, у которого есть имя и адрес в памяти, и к нему можно обращаться повторно.

## Rvalue
rvalue — временное значение, которое не имеет имени и живёт только в выражении.
```
int y = 5 + 3;   // результат 5+3 — это rvalue
int z = 42;      // литерал 42 — rvalue
```

Однако, **rvalue тоже существует в памяти** – процессору же нужно где-то хранить это значение, пока оно участвует в вычислениях.
Разница в том, что у rvalue:
- нет имени в языке C++
- нет стабильного адреса, на который можно сослаться (адрес может быть, но язык не даёт на него ссылку)

Где именно хранится rvalue зависит от ситуации:
- В регистрах процессора. Например:
    ```
    int y = 2 + 3;
    ```
    Здесь `2+3` может вычислиться прямо в регистре CPU, без выделения памяти в стеке.
- В стеке. Если результат временного вычисления нужно передать дальше, компилятор создаст временный объект в стеке:
    ```
    std::string s = std::string("hi") + " world";
    ```

❗ ***Но внутри функции параметр всегда `lvalue`, даже если его тип `T&&`.***
```
void foo(std::string&& s) {
    bar(s); // !!! s — это lvalue, даже если пришёл rvalue
}
```
Да, тип `s` это `std::string&&`, но само имя `s` это lvalue, потому что у него есть имя и адрес, и мы можем к нему обратиться повторно. Поэтому без `std::move(s)` или `std::forward` вызов `bar(s)` будет восприниматься как передача lvalue, и сработает копирование.

## Ссылки на lvalue и rvalue
Ссылка на lvalue (`T&`): может привязаться только к lvalue (исключение с `const&`). Ссылка на rvalue (`T&&`): может привязаться только к rvalue.

rvalue-ссылка — это способ сказать, что объект можно перемещать, «забирать» его ресурсы.
```
void foo(std::string&& s) {
    std::cout << s;
}
```

# `std::move`
Это просто каст в rvalue-ссылку:
```
std::string s = "Hello";
std::string t = std::move(s); // s превращается в rvalue, t может его переместить
```

# perfect forwarding
Прямая передача — это приём, когда функция передаёт аргумент дальше, сохраняя его значение (`lvalue` или `rvalue`). Обычно используется с шаблонами:
```
template <typename T>
void wrapper(T&& arg) {
    func(std::forward<T>(arg)); // arg передаётся как lvalue или rvalue, как было изначально
}
```
`std::forward<T>` — это «умная» версия `std::move`. Без `forward` аргумент внутри шаблона всегда будет `lvalue`, даже если передан `rvalue`. **То есть прямая передача позволяет сохранить семантику перемещения при шаблонных функциях.**

Пример:
```
void process(std::string& s) { std::cout << "lvalue\n"; }
void process(std::string&& s) { std::cout << "rvalue\n"; }

template <typename T>
void wrapper(T&& arg) {
    process(std::forward<T>(arg)); 
}

std::string s = "hi";
wrapper(s);                 // lvalue
wrapper(std::string("hi")); // rvalue
```

## Когда `std::forward` НЕ нужен

***Аргумент принимается по значению***
```
template<typename T>
void f(T x)
{
    foo(x);
}
```

`x` уже копия. Информация о `rvalue/lvalue` потеряна.


***Аргумент принимается по const&***
```
template<typename T>
void f(const T& x)
{
    foo(x);
}
```
`x` всегда lvalue.


***Аргумент не передаётся дальше***
```
template<typename T>
void print(T&& x)
{
    std::cout << x;
}
```




# `push_back` vs `emplace_back`
```
std::vector<std::string> v;
std::string s = "Hello";

// push_back
v.push_back(s);          // копирование
v.push_back(std::move(s)); // перемещение

// emplace_back
v.emplace_back("World"); // конструирует объект прямо в векторе
```
`push_back` принимает готовый объект и копирует или перемещает его в контейнер. 

Компилятор сначала должен сделать объект типа элемента вектора (например, Resource(5) или std::string("5")). Этот объект создаётся во временной памяти (обычно на стеке). `push_back` принимает этот объект по ссылке (или `rvalue`-ссылке) и копирует/перемещает его в память вектора.

Временный объект живёт отдельно (в стеке или оптимизирован в регистр/память компилятором). Потом он перемещается в область памяти, выделенную вектором.

`emplace_back` строит объект прямо внутри контейнера, минуя копирование/перемещение, что эффективнее для сложных объектов. Сразу вызывает конструктор элемента прямо в своей памяти, т.е. в заранее выделенном месте под элемент.

Технически внутри emplace_back примерно так (упрощённо):
```
template <class... Args>
T& emplace_back(Args&&... args) {
    if (size == capacity) grow();
    // вызвать конструктор T прямо в памяти вектора
    new (data + size) T(std::forward<Args>(args)...);
    ++size;
    return back();
}
```

## Когда использовать `push_back`

Если объект уже существует.

```
std::vector<std::string> v;

std::string s = "hello";
v.push_back(s);
```

или

```
v.push_back(std::string("hello"));
```

Сигнатура примерно такая:

void push_back(const T& value);
void push_back(T&& value);

## Когда использовать `emplace_back`

Когда нужно создать объект прямо внутри контейнера.

```
std::vector<std::string> v;

v.emplace_back("hello");
```

Контейнер получает аргументы конструктора и вызывает конструктор прямо в памяти контейнера.



# `const T&`
```
int main() {
    int a = 10;             // (1)
    const int& ref = a;     // (2)
    const int& ref1 = 42;   // (3)
    int b = a + ref1;       // (4)
}
```

`const int& ref1 = 42;`

у rvalue (например, у литерала 42) всегда где-то есть физическая память, если нужно привязать к нему ссылку. Для `const&` компилятор делает временный объект. Поэтому в (3) реально происходит скрытое создание переменной.

По стандарту C++ нельзя привязать обычную ссылку `int&` к rvalue, но можно привязать `const int&`. Чтобы это работало, компилятор создаёт временный объект в стеке (обычно прямо в `main`).
В этот временный кладётся 42. Ссылка `ref1` привязывается к этому временному объекту.

в этом примере получается, что и a, и тот «временный объект» для 42 живут в стеке и мало чем отличаются с точки зрения физической памяти. Но есть важная разница на уровне языка C++:
1. Переменная `a` это lvalue. У неё есть имя. У неё есть чётко определённое время жизни (с момента входа в `main` и до выхода). Ты можешь менять `a`, брать `&a` и передавать куда угодно.
2. «Временный объект» для 42 (`через const int& ref1 = 42;`) это rvalue (литерал), но стандарт разрешает привязать его к `const&`. Компилятор вынужден материализовать его как временную переменную (часто тоже на стеке, иногда даже может оптимизировать в регистр, если не нужен адрес). У него нет имени в исходном коде (у тебя он доступен только через `ref1`). Время жизни особенное: без ссылки — жил бы «только в выражении»; со ссылкой — время жизни продлено до времени жизни ссылки (`ref1`).
